# 资深 Agent 算法工程师视角：Solidity 终审（2026-05-28）

> 视角：带过团队、面过很多人、做过 agent 上线的 staff/资深算法工程师。结论先行，再拆解。

## 一句话结论（2026-05-28 第三次更新）
当前形态 **8.3 / 10**（含 APR-CS 创新尝试 + 500 工单 LLM 生成的规模化压测）。作为**简历/作品集**已超出
绝大多数候选人，处于"测过的 pre-pilot 候选"档位；作为**企业上线项目**仍是 prototype（差 3-6 个月工程化）。

## 企业落地可行性（500 工单压测真实数据驱动的诚实判断）
- **能上 pre-pilot？** 可以（带人工兜底）：guardrail 真在拦注入(17% 硬 block)/PII(22% 硬 block)；
  agent 不知道就转人工(85%)而不是乱编——**安全>过度自信**的设计意图被真实分布验证。
- **能上 GA？** 不行。500 工单压测暴露的 gap 全是上线前必补的：
  1. **KB 覆盖严重不足**：合成 KB 30 篇 → 实际话题千奇百怪 → easy 也只 23% 解决。
     上线前需把 KB 扩到覆盖真实业务 + 接 web/工单库 RAG。
  2. **类别路由缺失**：multi_intent / multilingual 几乎全转人工，需要前置 splitter / translator。
  3. **运维**：QPS 5.25（API 限速）、err 3.8%（无重试中间件）、p95 5.2s（无缓存）——都是单点工程未做。
  4. **记忆 1k 拐点**：BM25 + jsonl 在 1k 量级前可用，5k 已 48ms，必须接向量库或 TTL。

## 终版数据（已跑完，可直接放简历）
- 合成受控基准：解决率 34.2%→71.1%、keypoint 覆盖 42.3%→80%、重复错误率 100%→40%、转人工 F1 0→0.71；
  static 基线全程持平（清晰归因）。
- **τ²-bench retail，DeepSeek-V4-Flash，官方 pass^k（test=40×trials=4，160 sims/条件）**：
  OFF pass^1=0.925 → ON pass^1=0.931（Δ+0.006）；pass^2/^3 同向正增益（+0.008/+0.006）。
  Δ magnitude 受 model ceiling 限制（OFF 已 92.5%），方向稳健。
- **τ²-bench airline（最难域，更大 headroom），同模型，test=20×trials=4**：
  pass^1 OFF 0.800 → ON 0.775（Δ−0.025），但 **pass^2 +0.008、pass^3 +0.012**——
  揭示 "playbook 牺牲单次最优换多次一致性" 的真实 tradeoff（reliability scaffolding 视角下符合预期）。
- **APR-CS 改进尝试**（针对 airline tradeoff，实做不是空说）：
  实现 Adaptive Playbook Router + Counterfactual Tip Attribution（GEPA/Self-RAG/Mem0/Voyager/AlphaEvolve 综合借鉴）。
  合成集 4 条件消融：硬注入 75% → cf_weighted 100%、avg_tips 10→2.75；
  τ² airline cf_weighted：pass^1 +1.2pp 回升、但 pass^2/^3/^4 下降——发现 K=4 紧约束下是 tradeoff 不是 Pareto。
  下一步：adaptive-K / confidence-gated（需 agent 暴露 confidence 信号，受 tau2 接口限制留作下一阶段）。
  **这条"做了改进→暴露新 tradeoff→给出下一阶段研究问题"的闭环比"刷了指标"对资深更有说服力。**
- 弱监督消融：噪声隐式反馈下最终解决率达 gold 的 **92.6%**（8 seed 均值）。
- 检索消融：BM25/TF-IDF余弦/Hybrid 三种检索器进化增益方向**一致**（+0.37/+0.21/+0.40）。
- ROI 分析：解决率 34→71% 时 token +27%、延迟 +71%，**边际收益拐点在经验池≈17**。

## 强项（面试可主动展开的）
1. **双层证据**：合成集做*受控消融*（static/episodic/full 三条曲线，干净归因）+ τ²-bench 做*真实背书*（pass^1 81%→94%）。这正是资深看重的"controlled study + recognized benchmark"组合。
2. **闭环完整**：数据→失败捕获→离线反思→playbook→检索生效→再评测，端到端跑得通、可复现、有量化。
3. **防作弊评测**：verifier 用 gold label，agent 看不到；eval 是 train 的 held-out 改写 → 提升是泛化非背题。这一条能挡住最常见的质疑。
4. **安全/治理是一等公民**：guardrail（注入/PII/groundedness/合规）+ playbook 发布门禁 + 灰度回滚 + 审计 + misevolution 防护。多数候选人只字不提安全，这里是亮点。
5. **工程素养**：分层清晰、零依赖可跑、65 个测试、可观测有 p50/p95 与成本、可插拔后端（mock/API/vLLM）。
6. **诚实**：局限、N 偏小、mock 数字与真实数字分开标注——资深反而加分。

## 仍存在的风险 / 会被追问到的软肋（要能接住）
1. **τ² 的 N 偏小（test=16）**：+12.5pt 只是 2 个任务的差异，统计上不稳。→ *好答案*：承认；给出"一条命令扩到 40 任务 + pass^k(trials=4)"的复现路径；说明受成本约束，趋势方向正确。**这是冲 8 分最该补的一刀**。
2. **检索是 BM25、记忆是 jsonl**：非向量化、非服务化。→ *答案*：刻意选零依赖以保证确定性/可复现；接口已抽象，换向量库/重排是工程替换不是架构改动。
3. **"自进化"≈ 高级 RAG/记忆复用，不动权重**：会被问"本质区别"。→ *答案*：定位就是 in-context 自进化（2026 工业界主线：Anthropic dreaming、Devin 经验复用）；与微调/RLHF 是**互补的不同时间尺度**，且更可解释、可回滚、零训练成本。
4. **反思/蒸馏靠 LLM，没真用户**：线上无 gold label 的进化只做了*机制*（serving 的隐式反馈→待复盘队列），没有真实流量验证长期效果。→ 这是"作品集 vs 生产"的根本边界，要坦诚。
5. **记忆膨胀/污染、成本随记忆增长**：实现了 TTL/去重/冲突消解的*机制*，但未在大规模下压测。
6. **单域、单语言、合成 KB**：泛化到多域/多语种/真实脏数据未验证。

## 要冲到 8+ 还差的 Top5（按性价比）
1. **τ² 扩规模**：retail 完整 40 任务 + pass^k(trials≥4)，最好再加一个 airline/telecom 域 → 把"趋势"变"统计可信"。（成本可控，最高优先）
2. **加一条在线/噪声反馈的端到端小实验**：用 serving 的隐式反馈（模拟点踩/reopen）驱动一轮 governed 进化，画出"弱信号也能进化"的曲线 → 直击"线上没 label 怎么办"。
3. **成本/延迟 vs 收益曲线**：展示记忆增长后的检索延迟与 token 成本，证明你算过 ROI（资深极看重）。
4. **向量检索消融**：BM25 vs 向量/混合检索各一版，证明结论不依赖某种检索。
5. **一页"生产部署架构图"**：把 serving+guardrail+obs+governance 画成一张真实部署拓扑（含人接管、灰度、监控告警）。

## 给求职定位的话术
> "这是一个**可复现的自进化客服 Agent 作品**：在受控基准上干净地证明机制、在公认基准 τ²-bench 上验证对真实 agent 有效，并按生产标准补齐了 guardrail、可观测、发布治理与 serving。它和真上线系统的差距我很清楚——主要在规模化真实流量、在线长期效果与大规模记忆压测——但核心机制和工程治理与 2026 工业界主线是一致的。"

这样讲：**懂机制 + 懂基准 + 懂生产 + 懂边界**，四样齐了，资深面试官会认为 solid。
